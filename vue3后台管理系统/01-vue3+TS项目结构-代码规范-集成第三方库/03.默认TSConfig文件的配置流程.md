## 默认TSConfig文件的配置流程

tsconfig.json

```json
{
  "extends": "@vue/tsconfig/tsconfig.web.json", // tsconfig.js文件继承于前面这个文件
  "include": ["env.d.ts", "src/**/*", "src/**/*.vue"], // 告诉TS哪些文件需要经过tsconfig.js编译，
  "compilerOptions": { // 下面两个配置，用于给vscode读取的
    "baseUrl": ".",
    "paths": {
      "@/*": ["./src/*"]
    }
  },

  "references": [ // 引入文件
    {
        // 为什么要引入这个文件？一般不推荐修改当前文件，推荐修改下面引入的文件
      "path": "./tsconfig.node.json"
    }
  ]
}

```

tsconfig.node.json

- 这个文件会合并到tsconfig.json文件

```json
{
  "extends": "@vue/tsconfig/tsconfig.node.json", // node SSR 相关的东西
    // 基于vite环境的一些东西
  "include": ["vite.config.*", "vitest.config.*", "cypress.config.*", "playwright.config.*"],
  "compilerOptions": {
    "composite": true, // 如果这个文件能被tsconfig.json文件合并，需要开启此
    "types": ["node"] // include这些东西需要基于node环境
  }
}
```
